## 1. DCS / VisualDCS — corpus frequency + attestation ✅ WIRED

- **Artifact:** [`kosha/data/frequency/lemma_frequency.tsv`](https://github.com/gasyoun/kosha/blob/main/data/frequency/lemma_frequency.tsv)
  — 83,277 SLP1-keyed lemmas: all-corpus count, grammar tag, rank, per-period counts
  (10 DCS period slots), Leonchenko coverage weight. Built from VisualDCS's M9
  `archive.sqlite` by
  [`build_frequency_layer.py`](https://github.com/gasyoun/kosha/blob/main/data/frequency/build_frequency_layer.py).
- **Rights:** upstream is Oliver Hellwig's Digital Corpus of Sanskrit (**CC BY**); the
  TSV is a derived aggregation. No restriction on reuse with attribution.
- **Wired feed:** [`src/data/corpus-frequency.json`](https://github.com/sanskrit-lexicon/csl-guides/blob/main/src/data/corpus-frequency.json)
  (top 2,000 lemmas by corpus rank + whole-corpus stats; 4,550,704 tokens counted over
  59,282 counted lemmas), regenerated by
  [`scripts/build-corpus-frequency.mjs`](https://github.com/sanskrit-lexicon/csl-guides/blob/main/scripts/build-corpus-frequency.mjs)
  (`npm run build:corpus-frequency`; needs the sibling `kosha` checkout — the committed
  JSON keeps the site build sibling-free).
- **Prior art honored:** csl-atlas already consumes the *banded* view
  ([`csl-atlas/data/dcs/dcs_lemma_summary.json`](https://github.com/sanskrit-lexicon/csl-atlas/blob/main/data/dcs/dcs_lemma_summary.json),
  DCS-2021, frequency bands only). The guides feed instead consumes the richer 2026
  kosha layer (exact counts + per-period vectors) because the planned guides
  visualizations are diachronic. Nothing re-parsed from CoNLL-U (SHARED_CODE §8).

## 2. Heritage / INRIA (Gérard Huet) — independent lexicon/coverage layer ✅ WIRED

- **Artifact:** [`SanskritLexicography/HeadwordLists/mw_heritage_crosswalk.tsv`](https://github.com/gasyoun/SanskritLexicography/blob/master/HeadwordLists/mw_heritage_crosswalk.tsv)
  — entry-level MW↔Heritage crosswalk (H099, 03-07-2026): 185,803 MW SLP1 keys,
  25,140 Heritage-covered (13.5%), anchors into the Heritage `DICO/` hypertext
  dictionary.
- **Rights:** built from the
  [darkone23/Heritage_Resources](https://github.com/darkone23/Heritage_Resources)
  GitHub mirror (03-2025, **LGPLLR**). Redistribution of the mirror's *raw data* is a
  pending `@DECIDE` (GTD); the wired feed carries **aggregate statistics over our own
  derived crosswalk only**, which does not exercise that question.
  ⚠️ [sanskrit.inria.fr](https://sanskrit.inria.fr) and gitlab.inria.fr are Anubis
  bot-walled ([FINDINGS §41/§47](https://github.com/gasyoun/SanskritLexicography/blob/master/FINDINGS.md))
  — never fetch live.
- **Wired feed:** [`src/data/heritage-coverage.json`](https://github.com/sanskrit-lexicon/csl-guides/blob/main/src/data/heritage-coverage.json)
  (overall + per-SLP1-initial coverage, plus a 200-lemma sample joining the DCS
  top-frequency lemmas to their Heritage anchors), regenerated by
  [`scripts/build-heritage-coverage.mjs`](https://github.com/sanskrit-lexicon/csl-guides/blob/main/scripts/build-heritage-coverage.mjs)
  (`npm run build:heritage-coverage`; needs the sibling `SanskritLexicography`
  checkout).
- **Prior art honored:** coverage study + VH↔SLP1 bridge already delivered in
  [Huet-INRIA-Wordlist-vs-Cologne.md](https://github.com/gasyoun/SanskritLexicography/blob/master/HeadwordLists/Huet-INRIA-Wordlist-vs-Cologne.md);
  staged deeper reuse (frequency TSVs, morphology banks, segmenter-as-service) is
  owned by [HERITAGE_INRIA_ROADMAP.md](https://github.com/gasyoun/SanskritLexicography/blob/master/HERITAGE_INRIA_ROADMAP.md)
  Phases 3–6 — not duplicated here.

## 4. DharmaMitra (Berkeley) — MT/morphology ecosystem ⏳ RECORD ONLY

- **Touchpoints (registered in
  [PROJECT_INTERLINKS.md](https://github.com/gasyoun/Uprava/blob/main/PROJECT_INTERLINKS.md)):**
  supplier of GPU morphology/lemma runs feeding csl-atlas #89/#92; prospective

- **Rights:** the dharmamitra.org services expose no bulk-download license; their

  **Verdict: no direct feed.** The guides reach DharmaMitra-derived data only through
  the csl-atlas committed artifacts, and describe the ecosystem in prose (Stream 3)
  with links, not mined data.
