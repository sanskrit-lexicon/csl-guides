## üßÝ Dev Notes & Hypotheses (Bugs, ideas, context)

- **CDSL entry ids: extract OFFLINE from the local source, not the API.** `getword_xml.php`
  returns `<L>`(lnum) + `<pc>`(page,col) but THROTTLES hard (HTTP 429 after a short burst, plus
  TLS handshake timeouts) ‚Äî unusable for 40+ words. The same data is in the sibling repo
  `csl-orig/v02/mw/mw.txt`: each record is `<L>{id}<pc>{page,col}<k1>{slp1}‚Ä¶`. Index
  `k1 ‚Üí (first non-stub lnum, pc)` and read it directly. Every MW print page+col in the quiz
  matched the digital `<pc>` exactly. (Scratchpad: `local_remap.py`, `inject_dhatu.py`.)
  - **Skip cross-reference stubs.** A headword can appear twice ‚Äî a stub ("&c. See p. 1303",
    "See next p.", `<pcol>` tag) and the substantive entry. Match the stub regex and take the
    first NON-stub record, or you point at the wrong page (hetu, yoga, prak·πõti each had this).
  - **IAST‚ÜíSLP1** (multi-char aspirates/diphthongs first; strip hyphens for compound keys) lives

    fix: kƒÅla-yuga‚Üí**kali-yuga**, p≈´rva-pak·π£a-pada‚Üí**p≈´rva-pak·π£a-pƒÅda**.

- **CORS spike (2026-06-17, verified against the LIVE server, not just the doc).** Native CDSL
  API at `/scans/awork/apidev/{action}.php` sends `Access-Control-Allow-Origin: *` on every
  endpoint ‚Üí a static GitHub Pages page can `fetch` it cross-origin with **no proxy and no
  browser-UA spoof** (unlike the front page, which 403s non-browser UAs). Verdict: client-side
  fetch+parse (not iframe, not proxy). Use `getword.php` (LIVE, HTML entry ‚Äî keep inner
  `#CologneBasic`, sanitize) + `getsuggest.php` (LIVE JSON) for per-dict key resolution.
  - **The real problem is cross-dict headword forms, not CORS**: same lemma keys differently
    (MW stem `agni`; Apte nominative `agniH` ‚Äî matches the old `fix/ap90-screenshot-query`).
    `getsuggest(term)` returns prefix matches in the input scheme ‚Üí pick exact else first
    candidate (agni‚Üíap90 gives `agniH`). This is the resolver.
  - **`dalglob` REJECTED**: the experimental key‚Üídicts index (`dalglob.php`) is too flaky for a
    UI ‚Äî 20‚Äì40 s hangs, intermittent TLS handshake failures, and returned `{"status":404,
    "dicts":[]}` for `agni`. Doc already flags it experimental. Don't depend on it.
  - **Server flakiness is real**: rapid sequential hits sometimes time out / TLS-fail even on
    `getword`/`getsuggest` that work moments earlier ‚Üí fetch each column in parallel with its
    own timeout + error/retry state; never a tight sequential loop.
  - `getword`/`getsuggest`/`getword_xml` all CORS-OK; `getword_xml` is alpha (avoid). Input
    scheme is flexible (`input=deva` worked). Payloads vary wildly (5 KB MW vs 82 KB Apte).
