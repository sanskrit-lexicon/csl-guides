## рҹ§  Dev Notes & Hypotheses (Bugs, ideas, context)

- **CDSL entry ids: extract OFFLINE from the local source, not the API.** `getword_xml.php`
  returns `<L>`(lnum) + `<pc>`(page,col) but THROTTLES hard (HTTP 429 after a short burst, plus
  TLS handshake timeouts) вҖ” unusable for 40+ words. The same data is in the sibling repo
  `csl-orig/v02/mw/mw.txt`: each record is `<L>{id}<pc>{page,col}<k1>{slp1}вҖҰ`. Index
  `k1 вҶ’ (first non-stub lnum, pc)` and read it directly. Every MW print page+col in the quiz
  matched the digital `<pc>` exactly. (Scratchpad: `local_remap.py`, `inject_dhatu.py`.)
  - **Skip cross-reference stubs.** A headword can appear twice вҖ” a stub ("&c. See p. 1303",
    "See next p.", `<pcol>` tag) and the substantive entry. Match the stub regex and take the
    first NON-stub record, or you point at the wrong page (hetu, yoga, prakб№ӣti each had this).
  - **IASTвҶ’SLP1** (multi-char aspirates/diphthongs first; strip hyphens for compound keys) lives
    in `local_remap.py`. Two Wikner transliterations were wrong and the digital page confirmed the
    fix: kДҒla-yugaвҶ’**kali-yuga**, pЕ«rva-pakб№Јa-padaвҶ’**pЕ«rva-pakб№Јa-pДҒda**.
- **`Quiz` component (generalized in #38):** renders a quiz dataset as native `<details>` cards
  (no JS, SSR-safe, a11y). Takes a `data` prop (defaults to `mw-quiz.json`) + filters `lesson` /
  `group` / `types`; types include `lookup`, `trace-to-dhatu`, `convert`, `concept`,
  `multiple-choice`. Two datasets now: `mw-quiz.json` (MW) and `translit-quiz.json` (encoding).
  Add a new quiz = new JSON + `<Quiz data={вҖҰ} group="вҖҰ" />`; no component change needed.
- **Apte editions are three distinct dicts** (verified, source + catalog): `AP90` (1890 1st ed.,
  1196 pp), **`AP`** (Revised & Enlarged 1957вҖ“1959, eds. Gode & Karve, 1768 pp вҖ” repo `AP`, source
  `csl-orig/v02/ap`), both SanskritвҶ’English; **`AE`/`ApteES`** is the separate *Student's
  EnglishвҶ’Sanskrit* dict. **AP markup quirks vs AP90:** sense numbers use a **`вҲҷВІ`** marker
  (`вҲҷВІ1`, `вҲҷВІ2`, вҖҰ); page-columns are **numeric** (`<pc>0002-1`) not lettered (`0001-c`);
  grammatical label is `{%<lex>a.</lex>%}`. Shared Apte markup: `{#вҖҰ#}` SLP1, `{%вҖҰ%}` italic,
  `{@вҖҰ@}` bold, `<ls>` source. No Vedic accents.
- **Browser-only bug class (from the comparison-view spike):** `[...el.attributes]` (spreading a
  live `NamedNodeMap`) transpiles unreliably under Babel/minification вҖ” in the prod bundle items
  lost `.name`. Fix: `el.getAttributeNames()` + `el.getAttribute(name)`. `npm run build` green
  вү  runtime-correct; verify interactive widgets in a real browser against the BUILT bundle.
- **Sandbox can't browser-fetch the Cologne API (#42 verification, 2026-06-24).** Playwright
  chromium here loads github.io fine but an in-page `fetch` to `sanskrit-lexicon.uni-koeln.de`
  returns "Failed to fetch", while `curl` from the shell gets 200 + `ACAO:*` (CORS is fine). So
  the headless browser's egress to that host is blocked in this environment вҖ” interactive widgets
  that hit the live API can be SELECTOR/SSR-verified here but NOT entry-render-verified. (Don't
  misread it as a CORS or throttle bug; real users' browsers reach it.) Also: the API 429s hard
  under bursts вҖ” a 5-column compare вүҲ 10 near-simultaneous requests can self-throttle.

- **CORS spike (2026-06-17, verified against the LIVE server, not just the doc).** Native CDSL
  API at `/scans/awork/apidev/{action}.php` sends `Access-Control-Allow-Origin: *` on every
  endpoint вҶ’ a static GitHub Pages page can `fetch` it cross-origin with **no proxy and no
  browser-UA spoof** (unlike the front page, which 403s non-browser UAs). Verdict: client-side
  fetch+parse (not iframe, not proxy). Use `getword.php` (LIVE, HTML entry вҖ” keep inner
  `#CologneBasic`, sanitize) + `getsuggest.php` (LIVE JSON) for per-dict key resolution.
  - **The real problem is cross-dict headword forms, not CORS**: same lemma keys differently
    (MW stem `agni`; Apte nominative `agniH` вҖ” matches the old `fix/ap90-screenshot-query`).
    `getsuggest(term)` returns prefix matches in the input scheme вҶ’ pick exact else first
    candidate (agniвҶ’ap90 gives `agniH`). This is the resolver.
  - **`dalglob` REJECTED**: the experimental keyвҶ’dicts index (`dalglob.php`) is too flaky for a
    UI вҖ” 20вҖ“40 s hangs, intermittent TLS handshake failures, and returned `{"status":404,
    "dicts":[]}` for `agni`. Doc already flags it experimental. Don't depend on it.
  - **Server flakiness is real**: rapid sequential hits sometimes time out / TLS-fail even on
    `getword`/`getsuggest` that work moments earlier вҶ’ fetch each column in parallel with its
    own timeout + error/retry state; never a tight sequential loop.
  - `getword`/`getsuggest`/`getword_xml` all CORS-OK; `getword_xml` is alpha (avoid). Input
    scheme is flexible (`input=deva` worked). Payloads vary wildly (5 KB MW vs 82 KB Apte).
